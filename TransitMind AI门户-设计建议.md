# TransitMind AI 业务用户门户 - Demo 阶段设计建议

> 面向政府国企内网部署的智能体平台优化方案

## 一、整体定位与策略

### 1.1 Demo 阶段核心目标

Demo 阶段应聚焦于验证核心价值主张，而非追求功能完整性：

```
核心价值验证优先级：
┌─────────────────────────────────────────────────────┐
│  P0: 自然语言驱动的工作模型（核心创新）               │
│  P1: 智能体超市 - 选择适合的智能体应用                │
│  P2: 知识库配置 - 让智能体懂你的业务                  │
└─────────────────────────────────────────────────────┘
```

### 1.2 核心产品模型

```
用户工作流程：
┌─────────────────────────────────────────────────────────────┐
│                                                             │
│   用户输入自然语言                                           │
│        ↓                                                    │
│   ┌─────────────────┐                                       │
│   │ 智能交互中枢     │  ← 思考、规划、理解意图               │
│   │ (自然语言向导)   │                                       │
│   └────────┬────────┘                                       │
│            ↓                                                │
│   ┌─────────────────┐                                       │
│   │ 智能体超市       │  ← 推荐或跳转至合适的智能体           │
│   │ (智能体应用市场) │                                       │
│   └────────┬────────┘                                       │
│            ↓                                                │
│   ┌─────────────────┐                                       │
│   │ 智能体工作空间   │  ← 执行任务、输出结果                 │
│   │ (沉浸式交互)     │                                       │
│   └─────────────────┘                                       │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 1.3 目标用户画像

政府国企用户具有以下特点，设计需针对性优化：

| 特征 | 设计影响 |
|------|----------|
| 年龄层偏大 | 界面字体偏大、动效可关闭、操作路径短 |
| 对新系统有学习成本顾虑 | 新手引导、操作提示、传统视图备选 |
| 权限敏感 | 完善的角色管理、操作审计、数据脱敏 |
| 内网环境 | 离线能力、浏览器兼容、资源本地化 |

### 1.4 建议的实施策略

**做减法**：聚焦核心模块做透，权限配置收纳到设置

**推荐 Demo 范围**：

| 模块 | Demo 阶段 | 完整版 |
|------|-----------|--------|
| 智能交互中枢（自然语言向导） | ✅ 核心实现 | 扩展意图识别能力 |
| 智能体超市 | ✅ 核心实现 | 完整订阅/评论/统计 |
| 智能体工作空间 | ⚡ 简化版 | 完整编辑/协作能力 |
| 知识库配置 | ✅ 亮点功能 | 多知识库管理 |
| 系统设置（含权限） | ⚡ 基础版 | 完整权限矩阵 |

## 三、用户体验设计建议

### 3.1 交互设计原则

```
政府国企用户友好原则：
┌─────────────────────────────────────────────────────┐
│  1. 自然语言优先：用户只需说话，系统理解并执行        │
│  2. 稳定性优先：动效可关闭，降级方案完备              │
│  3. 易学性优先：新手引导、操作提示、帮助文档          │
│  4. 效率优先：常用功能快捷入口、键盘快捷键            │
└─────────────────────────────────────────────────────┘
```

### 3.2 自然语言交互规范

```
输入引导：
├── 默认提示语：「有什么可以帮您的？」
├── 示例推荐：根据用户角色推荐常用指令
└── 历史记录：快速选择历史输入

响应规范：
├── 思考过程可视化（增强信任感）
├── 明确告知下一步操作
├── 提供多个选项时说明差异
└── 执行失败时给出建议
```

### 3.3 视觉模式

| 模式 | 适用场景 | 特点 |
|------|----------|------|
| 标准模式 | 年轻用户、演示场景 | 完整动效、科技感设计 |
| 简洁模式 | 日常办公、老旧设备 | 关闭动画、提升性能 |
| 护眼模式 | 长时间使用 | 浅色主题、降低对比度 |

### 3.4 新手引导设计

```
引导流程：
├── 首次登录
│   ├── 欢迎页面（产品核心价值介绍）
│   ├── 自然语言输入演示
│   └── 功能导览（可跳过）
├── 功能引导
│   ├── 首次使用自然语言输入
│   ├── 首次进入智能体超市
│   └── 首次配置知识库
└── 帮助系统
    ├── 内嵌帮助提示（? 图标）
    ├── 操作手册入口
    └── 常见问题 FAQ
```
