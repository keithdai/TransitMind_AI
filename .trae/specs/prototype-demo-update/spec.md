# 高保真原型修改与演示方案 Spec

## Why

根据《智能体市场场景设计方案》和《高保真原型修改方案》，需要对现有高保真原型进行同步修改，编制mock数据，梳理交互流程，确保每个功能都能进行演示，并输出演示操作手册。

## What Changes

- **Marketplace.tsx**：更新智能体列表、分类、详情页，添加演示场景
- **Workspace.tsx**：更新常用智能体、示例对话内容
- **Home.tsx**：更新快捷建议、常用智能体
- **Mock数据**：为14个智能体编制完整的演示数据
- **交互流程**：梳理每个智能体的交互逻辑和演示流程
- **演示操作手册**：输出完整的演示指南文档

## Impact

- Affected specs: 智能体市场、工作台、首页
- Affected code: 
  - src/pages/Marketplace.tsx
  - src/pages/Workspace.tsx
  - src/pages/Home.tsx
  - src/data/agents.ts (新建)

## ADDED Requirements

### Requirement: 智能体市场更新

系统应提供按照设计方案更新的智能体市场，包含MVP阶段7个智能体和T2阶段7个智能体。

#### Scenario: 用户浏览智能体市场
- **WHEN** 用户进入智能体市场页面
- **THEN** 系统显示分类清晰的智能体列表
- **AND** MVP阶段智能体标记为"已上线"
- **AND** T2阶段智能体标记为"即将上线"

#### Scenario: 用户查看智能体详情
- **WHEN** 用户点击某个智能体卡片
- **THEN** 系统显示智能体详情页
- **AND** 包含核心能力描述
- **AND** 包含演示场景对话示例

### Requirement: Mock数据编制

系统应为每个智能体提供完整的演示数据，确保功能可演示。

#### Scenario: 演示数据完整性
- **WHEN** 用户使用任意智能体
- **THEN** 系统返回预设的演示响应
- **AND** 响应内容与场景设计方案一致
- **AND** 包含完整的对话流程

### Requirement: 演示操作手册

系统应提供完整的演示操作手册文档。

#### Scenario: 演示指南完整性
- **WHEN** 用户查阅演示操作手册
- **THEN** 手册包含每个智能体的演示步骤
- **AND** 包含预期输入和输出
- **AND** 包含演示话术建议

## MODIFIED Requirements

### Requirement: 工作台常用智能体

工作台常用智能体应更新为政务办公相关智能体。

**修改内容**：
- 会议纪要生成
- 公文起草
- 政策解读
- 工作汇报
- 督查督办
- 请示批复

### Requirement: 首页快捷建议

首页快捷建议应更新为政务办公相关建议。

**修改内容**：
- "帮我生成一份会议纪要"
- "起草一份工作汇报材料"
- "解读最新的政策法规"
- "撰写一份请示公文"
