# 智能体市场场景设计方案与实现需求

## 设计理念

### 智能体 vs 传统信息化工具

| 维度   | 传统信息化工具   | 智能体        |
| ---- | --------- | ---------- |
| 交互方式 | 表单填写、按钮操作 | 自然语言对话     |
| 执行模式 | 用户驱动，被动响应 | 意图驱动，主动执行  |
| 决策能力 | 固定流程，无决策  | 自主判断，灵活调整  |
| 主动能力 | 无主动性      | 主动建议、预警、追问 |
| 学习能力 | 无学习能力     | 从交互中学习偏好   |
| 个性化  | 千人一面      | 因人而异的个性化服务 |

### 智能体核心能力模型

```
┌─────────────────────────────────────────────────────────┐
│                    智能体核心能力                        │
├─────────────────────────────────────────────────────────┤
│  意图理解    →  理解用户真实需求，识别隐含意图          │
│  主动澄清    →  不确定时主动追问，获取关键信息          │
│  自主规划    →  分解任务，规划执行步骤                  │
│  工具调用    →  自主选择并调用合适的工具                │
│  推理判断    →  基于知识进行推理，做出判断              │
│  主动建议    →  发现潜在问题，主动提出建议              │
│  上下文记忆  →  记住历史交互，理解上下文                │
│  持续学习    →  从反馈中学习，优化服务质量              │
└─────────────────────────────────────────────────────────┘
```

### 1. 知识库检索智能体

#### 1.1 智能体定位

**"我理解你要找什么，比你更懂你需要什么"**

不只是搜索工具，而是能够理解用户真实意图、主动推荐相关知识、发现知识关联的智能助手。

#### 1.2 核心智能能力

| 能力    | 描述            | 示例                      |
| ----- | ------------- | ----------------------- |
| 意图理解  | 理解搜索背后的真实需求   | 用户搜"报销"，智能体理解用户可能要报销差旅费 |
| 主动追问  | 信息不足时主动澄清     | "您是想了解报销流程，还是报销标准？"     |
| 关联推荐  | 发现知识间的隐含关联    | 找到制度后，主动推荐相关的审批流程       |
| 知识图谱  | 构建知识间的关联关系    | 政策→解读→案例→模板的关联链         |
| 个性化排序 | 根据用户角色调整结果优先级 | 财务人员看到财务制度优先            |

#### 1.3 对话式交互设计

### 2. 知识库问答智能体

#### 2.1 智能体定位

**"不只是回答问题，而是帮您理解政策、解决问题"**

不只是问答工具，而是能够深度理解政策意图、提供实操建议、预判问题风险的智能顾问。

#### 2.2 核心智能能力

| 能力   | 描述          | 示例                    |
| ---- | ----------- | --------------------- |
| 深度理解 | 理解政策背后的立法意图 | 解释为什么这样规定，不只是引用条文     |
| 实操指导 | 提供可执行的解决方案  | 不只说"按规定"，而是告诉具体怎么做    |
| 风险预警 | 预判可能的风险点    | "您这种情况可能存在XX风险，建议..." |
| 案例对照 | 对照类似案例的处理方式 | "去年有类似情况，处理方式是..."    |
| 追问引导 | 引导用户补充关键信息  | "您说的项目类型是？这会影响适用政策"   |

#### 2.3 对话式交互设计

**场景1：深度政策解读**

```
用户：项目采购金额达到多少需要公开招标？

智能体：根据《政府采购法》和《招标投标法》规定：

📊 **公开招标门槛标准**：
- 货物类：200万元以上
- 服务类：200万元以上
- 工程类：400万元以上

---
💡 **深度解读**：
这个标准的设定是为了平衡采购效率和公平竞争。低于门槛的项目可以选择竞争性谈判、询价等方式，效率更高。

⚠️ **特别提醒**：
1. 这里的"金额"是指单项或批量采购金额
2. 不得故意拆分项目规避公开招标（这是违规行为）
3. 部分特殊项目可以申请变更采购方式

📋 **实操建议**：
您的项目预算是多少？我可以帮您判断：
- 适用哪种采购方式
- 需要准备哪些材料
- 审批流程是什么
```

**场景2：主动风险预警**

```
用户：我们想直接委托一家公司来做这个项目，预算150万

智能体：⚠️ **风险预警**

您提到的"直接委托"方式，预算150万，我需要提醒您注意以下风险：

🔴 **合规风险**：
- 150万的服务项目，已超过公开招标门槛（200万）的75%
- 直接委托需要满足法定情形，否则可能被认定为规避招标

📋 **直接委托的法定情形**：
1. 只能从唯一供应商处采购
2. 发生了不可预见的紧急情况
3. 必须保证原有采购项目一致性

💡 **建议方案**：
根据您的情况，我建议考虑以下合规方式：
- 竞争性谈判：时间较短（约20天），竞争性较强
- 竞争性磋商：可以综合评审，不只是看价格

需要我帮您分析哪种方式更适合吗？
```

**场景3：多轮对话解决问题**

```
用户：我们有个项目想变更采购方式

智能体：好的，采购方式变更需要谨慎处理。让我先了解一下情况：

📌 请问您目前的采购方式是？
1. 公开招标
2. 邀请招标
3. 竞争性谈判
4. 其他

用户：公开招标，但是两次流标了

智能体：明白了，公开招标两次流标是变更采购方式的常见情形。

📋 **处理流程**：
1. 分析流标原因（投标人不足？报价超预算？）
2. 报财政部门审批
3. 变更为竞争性谈判或竞争性磋商

🔍 **请问流标原因是什么？**这会影响后续处理：
- 投标人不足：可能需要调整资格条件
- 报价超预算：可能需要调整需求或预算

用户：两次都只有一家公司来投标

智能体：这种情况建议按以下步骤处理：

📋 **处理建议**：

**第一步：分析原因**
- 资格条件是否过高？
- 项目是否有特殊性？
- 市场竞争是否充分？

**第二步：准备变更材料**
- 招标公告及投标情况说明
- 流标原因分析报告
- 拟变更的采购方式及理由

**第三步：报批**
- 报财政部门审批
- 预计审批时间：5个工作日

💡 **我注意到**：只有一家公司投标，可能说明这家公司对这个项目有特殊优势。如果确实只能从唯一供应商采购，也可以考虑直接委托，但需要充分论证。

需要我帮您起草变更申请吗？
```

#### 2.4 智能体工作流程

```
用户提问 → 问题分类 → 复杂度判断 →
    ↓
简单问题 → 直接检索 → 生成答案 → 来源引用
    ↓
复杂问题 → 意图分析 → 信息完整性判断 →
              ├─ 信息完整 → 深度分析 → 生成答案 → 风险预警 → 实操建议
              └─ 信息不完整 → 追问 → 收集信息 → 继续分析
                                          ↓
                                    用户反馈学习
```

### 3. 智能公文助手

#### 3.1 智能体定位

**"不只是生成公文，而是帮您理清思路、把好关口"**

不只是公文模板工具，而是能够理解公文意图、主动补充内容、检查合规风险、提供修改建议的智能写作助手。

#### 3.2 核心智能能力

| 能力   | 描述            | 示例                      |
| ---- | ------------- | ----------------------- |
| 意图理解 | 理解用户为什么要写这个公文 | 用户说"申请"，智能体判断是请示还是报告    |
| 内容推理 | 根据零散信息推理完整内容  | 用户只说"买电脑"，智能体推理背景、理由、预算 |
| 风格适配 | 根据公文类型调整语言风格  | 请示要恳切，通知要明确，函要礼貌        |
| 合规检查 | 实时检查内容合规性     | "这个表述可能存在风险，建议修改为..."   |
| 智能润色 | 提升公文的专业性和可读性  | "这段表述可以更精炼，建议..."       |

#### 3.3 对话式交互设计

**场景1：从零开始的公文创作**

```
用户：帮我写个请示

智能体：好的，我来帮您起草请示。

📌 请示一般用于向上级机关请求批准事项。请告诉我：

**您想请示什么事项？**
例如：
- 申请经费
- 申请采购
- 申请人员编制
- 申请项目立项
- 其他事项

简单描述一下您的需求，我来帮您整理思路、起草正文。

用户：想申请采购一批办公设备

智能体：明白了，采购办公设备的请示。让我帮您梳理一下：

📝 **请示要素清单**：
✅ 请示事项：采购办公设备
❓ 背景原因：为什么需要采购？（设备老化？新增人员？）
❓ 具体内容：采购什么设备？数量？预算？
❓ 资金来源：预算从哪里出？

---
**请您补充以下信息**（可以随意说，我来整理）：
1. 为什么要采购？（简单说几句）
2. 需要采购哪些设备？
3. 大概多少钱？

用户：我们科室来了两个新人，现有电脑不够用，想买2台电脑，预算大概1万5

🤖 智能体：好的，信息已收集完整。让我为您起草请示：

**关于申请采购办公设备的请示**

    妥否，请批示。

                                                    XX科室
                                                  2026年3月24日

💡 **智能建议**：
1. ✅ 结构完整，符合请示格式要求
2. ⚠️ 建议：可以补充电脑的具体配置要求
3. ⚠️ 建议：可以说明期望的采购时间

需要我帮您：
```

### 3.2 交互设计原则

1. **自然语言优先**：用户不需要学习操作，直接说话
2. **主动引导**：智能体主动提问、主动建议
3. **渐进式交互**：从简单开始，逐步深入
4. **透明可解释**：让用户知道智能体在做什么、为什么这样做
5. **可控可干预**：用户随时可以干预、修改智能体的行为

### 3.3 实现技术架构

```
用户输入 → 意图识别引擎 → 对话管理器 → 任务规划器 →
                                    ↓
                            工具调用引擎 → 知识检索 → 内容生成 →
                                    ↓
                            质量检查 → 风险评估 → 输出响应
                                    ↓
                            用户反馈 → 学习优化
```
