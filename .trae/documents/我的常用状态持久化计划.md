# "我的常用"智能体状态持久化计划

## 需求分析

当前首页的"我的常用"智能体列表存在以下问题：
- 每次刷新页面或重新部署后，列表会重置为默认值（MVP阶段前6个智能体）
- 用户自定义添加/删除的智能体无法保留

### 功能要求
- 保存用户自定义的常用智能体列表
- 刷新页面后保留用户配置
- 重新部署后保留用户配置

## 实现方案

### 1. 数据存储

使用 localStorage 存储用户自定义的常用智能体ID列表：
- Key: `myFavoriteAgents`
- Value: 字符串数组（智能体ID列表）

### 2. 修改文件

**文件：`src/pages/Home.tsx`**

#### 2.1 初始化状态时读取 localStorage

```typescript
const [myAgents, setMyAgents] = useState(() => {
  const saved = localStorage.getItem('myFavoriteAgents')
  if (saved) {
    const savedIds = JSON.parse(saved) as string[]
    const savedAgents = savedIds
      .map(id => availableAgents.find(a => a.id === id))
      .filter(Boolean) as typeof defaultAgents
    return savedAgents.length > 0 ? savedAgents : defaultAgents
  }
  return defaultAgents
})
```

#### 2.2 添加/删除智能体时同步保存

```typescript
useEffect(() => {
  const agentIds = myAgents.map(a => a.id)
  localStorage.setItem('myFavoriteAgents', JSON.stringify(agentIds))
}, [myAgents])
```

### 3. 实现步骤

#### Step 1: 修改 myAgents 初始化逻辑
- 从 localStorage 读取已保存的智能体ID列表
- 根据ID列表还原智能体对象
- 如果没有保存数据或数据无效，使用默认值

#### Step 2: 添加 useEffect 监听状态变化
- 当 myAgents 变化时，自动保存到 localStorage
- 保存的是智能体ID数组，而非完整对象

#### Step 3: 验证功能
- 添加/删除智能体后刷新页面，验证状态保留
- 清除 localStorage 后刷新，验证恢复默认值
