# 智能体超市扩展实施计划

## 目标
将 `政务办公AI智能体平台功能设计.md` 中的 28 个智能体添加到智能体超市，并完善 Mock 数据便于演示。

## 实施步骤

### 第一阶段：分析现有代码结构
1. 阅读 `src/pages/Marketplace.tsx` 了解现有智能体数据结构
2. 了解 Agent 接口定义和字段要求
3. 确认分类数据的存储方式

### 第二阶段：添加智能体数据
按照文档分类，添加以下智能体：

#### 2.1 材料生产类智能体（12个）
1. 会议纪要生成 - agent_meeting_minutes 📝
2. 请示文与公文起草 - agent_document_draft 📄
3. 招标采购需求文件与合同编写 - agent_procurement 📋
4. 周报月报收口 - agent_weekly_monthly 📊
5. 公文审核校对 - agent_document_review ✅
6. 会议材料提纲自动生成 - agent_outline 📑
7. 领导汇报PPT编制 - agent_ppt 📈
8. 年度总结与述职材料编制 - agent_annual_summary 📋
9. OA通讯稿自动生成 - agent_news 📰
10. 年报/简报编制 - agent_annual_report 📚
11. 领导讲话稿与发言提纲 - agent_speech 🎤
12. 报会办会材料生成与通知辅助 - agent_meeting_materials 🏢

#### 2.2 知识检索类智能体（5个）
1. 历史会议纪要检索 - agent_meeting_search 🔍
2. 政策法规问答 - agent_policy_qa ⚖️
3. 局内制度流程问答 - agent_process_qa 📋
4. 历史材料复用 - agent_material_reuse 📂
5. 单项目资料问答 - agent_project_qa 🗂️

#### 2.3 业务协同类智能体（8个）
1. 信访舆情自动筛选分发与答复辅助 - agent_petition 📩
2. 人大建议和政协提案处置辅助 - agent_proposal 🏛️
3. 督办事项分解与跟踪 - agent_supervision 🎯
4. 外业踏勘前内业支撑 - agent_field_work 🛣️
5. 建设项目台账管理与进展报告 - agent_project_management 🏗️
6. 领导批示件办理辅助 - agent_instruction 📌
7. 专项工作专班协同 - agent_special_work 👥
8. 新人入职与岗位辅导 - agent_onboarding 👋

#### 2.4 数据分析类智能体（3个）
1. 固投台账报表自动生成 - agent_investment_report 📈
2. 研究项目台账报表与节点提醒 - agent_research_project 🔬
3. 数据分析与图表说明 - agent_data_analysis 📊

### 第三阶段：完善每个智能体的 Mock 数据
为每个智能体完善以下字段：
- id: 唯一标识符
- name: 智能体名称
- description: 功能描述
- category: 分类
- icon: 图标（使用emoji）
- color: 渐变色
- rating: 评分（4.0-5.0）
- users: 使用人数（100-5000）
- tags: 标签数组
- subscribed: 订阅状态
- features: 功能特点数组
- knowledgeBases: 知识库数组
- useCases: 用例数组
- recommended: 是否推荐
- isExternal: 是否外部服务
- externalUrl: 外部服务地址（可选）

### 第四阶段：测试和验证
1. 运行 `npm run build` 确保无编译错误
2. 检查智能体超市页面显示效果
3. 验证搜索和筛选功能正常

### 第五阶段：提交代码
1. 提交更改到 Git
2. 推送到 GitHub
3. 部署到 Vercel

## 技术要点
- 使用 localStorage 持久化自定义智能体数据
- 保持与现有代码风格一致
- 确保 TypeScript 类型正确
