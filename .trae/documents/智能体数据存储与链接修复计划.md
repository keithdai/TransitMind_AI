# 智能体数据存储与链接修复计划

## 需求分析

用户反馈两个问题：

1. **数据存储问题**：自定义智能体存储在 localStorage 中，换电脑后数据丢失。希望保存在项目目录下，实现跨电脑共享。
2. **外部链接问题**：新上架的外部智能体点击"立即使用"时，链接前面会加一段IP，应该直接使用服务链接字段。

## 实现方案

### 方案1：一键导入/导出所有上架智能体

**功能说明：**

* **导出**：一键导出所有上架的自定义智能体为 JSON 文件，用户可以下载保存

* **导入**：一键导入 JSON 文件，覆盖当前的自定义智能体列表

* **同步**：不同设备可以通过导入 JSON 文件实现数据同步

**数据文件：**

* 创建 `public/data/custom-agents.json` 作为默认数据源

* 用户可以将导出的 JSON 文件提交到项目中，实现团队共享

### 方案2：修复外部链接

**问题排查：**

* 检查 `handleUse` 函数是否正确获取 `externalUrl`

* 确保 `selectedAgent` 包含完整的 `externalUrl` 信息

## 实现步骤

### Step 1: 创建默认数据文件

* 创建 `public/data/custom-agents.json` 文件

* 初始内容为空数组 `[]`

### Step 2: 添加导出功能

* 在智能体市场页面添加"导出"按钮

* 点击后下载所有自定义智能体为 JSON 文件

* 文件名格式：`custom-agents-YYYYMMDD.json`

### Step 3: 添加导入功能

* 在智能体市场页面添加"导入"按钮

* 点击后选择 JSON 文件

* 导入后覆盖当前的自定义智能体列表

### Step 4: 修复外部链接

* 确保 `handleUse` 函数正确使用 `externalUrl`

## 代码修改清单

| 文件                               | 修改内容             |
| -------------------------------- | ---------------- |
| `public/data/custom-agents.json` | 新建 - 默认数据文件      |
| `src/pages/Marketplace.tsx`      | 添加导入/导出功能、修复外部链接 |
