# 登录页面与底部版权信息实现计划

## 需求分析

用户需要：
1. 创建登录页面，进入系统前先登录
2. 登录页面包含 Logo 和系统名称
3. 假登录功能 - 无论用户名密码如何都可以通过
4. 登录页面和主页面底部都添加"深城交研究院研发"

## 实现方案

### 1. 创建认证上下文

**文件：`src/contexts/AuthContext.tsx`**

```typescript
interface AuthContextType {
  isAuthenticated: boolean
  login: () => void
  logout: () => void
}
```

- 使用 localStorage 存储登录状态
- 提供 login/logout 方法

### 2. 创建登录页面

**文件：`src/pages/Login.tsx`**

登录页面设计：
- 居中布局，科技感风格
- Logo + 系统名称（TransitMind AI）
- 用户名输入框
- 密码输入框
- 登录按钮
- 底部版权信息："深城交研究院研发"

### 3. 修改路由配置

**文件：`src/App.tsx`**

- 添加登录路由 `/login`
- 创建受保护路由组件，未登录时重定向到登录页

### 4. 修改 Layout 底部

**文件：`src/components/Layout.tsx`**

- 在 footer 中添加"深城交研究院研发"

## 实现步骤

### Step 1: 创建认证上下文
- 创建 `src/contexts/AuthContext.tsx`
- 提供 isAuthenticated 状态和 login/logout 方法
- 使用 localStorage 持久化登录状态

### Step 2: 创建登录页面
- 创建 `src/pages/Login.tsx`
- 科技感设计风格，与现有系统风格一致
- 包含 Logo、系统名称、登录表单
- 假登录逻辑 - 任意用户名密码都可登录
- 底部版权信息

### Step 3: 修改路由配置
- 修改 `src/App.tsx`
- 添加登录路由
- 创建 ProtectedRoute 组件保护主页面路由

### Step 4: 修改 Layout 底部
- 修改 `src/components/Layout.tsx`
- 在 footer 添加"深城交研究院研发"

### Step 5: 添加登出功能
- 在 Layout 的用户菜单中添加登出按钮
- 点击后清除登录状态并跳转到登录页
