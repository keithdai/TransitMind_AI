# 外部知识库链接功能实现计划

## 需求分析

用户希望在知识库管理页面顶部增加两个外部知识库入口：
1. **RAGFlow** - 外部知识库系统
2. **Dify** - 外部知识库系统

### 功能要求
- 可以配置链接地址
- 点击后在新窗口跳转到配置的外部链接
- 配置信息持久化存储

## 实现方案

### 1. 数据结构设计

```typescript
interface ExternalKnowledgeBase {
  id: string
  name: string
  icon: string
  url: string
  description: string
  color: string
}
```

### 2. 修改文件

**文件：`src/pages/KnowledgeBase.tsx`**

#### 2.1 添加外部知识库数据
- 定义默认的外部知识库配置（RAGFlow 和 Dify）
- 从 localStorage 读取用户自定义的链接配置

#### 2.2 添加外部知识库展示区域
- 在页面顶部、现有知识库列表上方添加外部知识库区域
- 使用卡片样式展示 RAGFlow 和 Dify
- 每个卡片包含：图标、名称、描述、配置按钮

#### 2.3 添加配置功能
- 点击配置按钮弹出配置弹窗
- 弹窗中可输入/修改链接地址
- 保存到 localStorage

#### 2.4 添加跳转功能
- 点击外部知识库卡片时，在新窗口打开配置的链接
- 如果未配置链接，提示用户先配置

### 4. 实现步骤

#### Step 1: 添加外部知识库数据结构

- 在知识库列表上方添加"外部知识库"区域
