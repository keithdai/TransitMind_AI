# 修复"我的常用"智能体点击跳转问题

## 需求分析

用户反馈：把上架的智能体添加到"我的常用"后点击，不是访问上架智能体的链接，而是跳转到固定的工作台页面。

## 问题分析

**当前代码逻辑：**

```typescript
const handleAgentClick = (agentId: string) => {
  navigate(`/workspace/${agentId}`)
}
```

问题在于 `handleAgentClick` 函数只跳转到工作台页面，没有检查智能体是否有外部链接。

**需要修改为：**

1. 获取智能体的完整信息（包括 externalUrl）
2. 如果有 externalUrl，打开外部链接
3. 如果没有，跳转到工作台页面

## 实现方案

### 修改文件：`src/pages/Home.tsx`

1. 从 localStorage 读取自定义智能体数据
2. 合并预设智能体和自定义智能体
3. 在 `handleAgentClick` 中查找智能体并检查 externalUrl

### 代码修改

```typescript
const handleAgentClick = (agentId: string) => {
  // 从自定义智能体中查找
  const customAgents = JSON.parse(localStorage.getItem('customAgents') || '[]')
  const allAgents = [...agents, ...customAgents]
  const agent = allAgents.find(a => a.id === agentId)
  
  if (agent?.externalUrl) {
    // 有外部链接，打开外部链接
    window.open(agent.externalUrl, '_blank')
  } else {
    // 没有外部链接，跳转到工作台
    navigate(`/workspace/${agentId}`)
  }
}
```

## 验收标准

* [ ] 预设智能体点击后跳转到工作台页面

* [ ] 有外部链接的自定义智能体点击后打开外部链接

* [ ] 没有外部链接的自定义智能体点击后跳转到工作台页面
