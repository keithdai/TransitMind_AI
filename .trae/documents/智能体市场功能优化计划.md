# 智能体市场功能优化计划

## 需求分析

用户反馈三个问题：
1. **Bug**：上架智能体后填写的链接，点击"立即使用"没有访问该链接
2. **新功能**：需要增加删除智能体的选项
3. **优化**：搜索智能体下面的分类太多，需要整合

## 问题分析

### 问题1：外部链接未生效
- 当前 `handleUse` 函数只是打开 `/workspace/${agent.id}` 页面
- 上架智能体时填写的 `externalUrl` 没有保存到 Agent 对象中
- Agent 接口缺少 `externalUrl` 字段

### 问题2：缺少删除功能
- 只能添加自定义智能体，不能删除
- 需要区分预设智能体和自定义智能体

### 问题3：分类过多
当前12个分类：
- 全部、知识检索、公文处理、会议管理、采购招标、项目管理、审核检查、造价管理、投标管理、人力资源、培训入职、外业踏勘

建议整合为5个分类：
- 全部、知识管理（知识检索+公文处理）、业务流程（会议管理+项目管理+审核检查）、采购招标（采购招标+造价管理+投标管理）、人力资源（人力资源+培训入职+外业踏勘）

## 实现方案

### 1. 修复外部链接功能

**修改文件：`src/data/agents.ts`**
- 在 Agent 接口中添加 `externalUrl?: string` 字段

**修改文件：`src/pages/Marketplace.tsx`**
- 保存自定义智能体时，保存 `externalUrl` 字段
- 修改 `handleUse` 函数：如果有 `externalUrl`，打开外部链接；否则打开工作台页面

### 2. 添加删除功能

**修改文件：`src/pages/Marketplace.tsx`**
- 在智能体详情页添加删除按钮
- 只对自定义智能体（id 以 'custom-' 开头）显示删除按钮
- 删除时从 `customAgents` 中移除

### 3. 整合分类

**修改文件：`src/data/agents.ts`**
- 修改 `categories` 数组，整合为5个分类
- 修改每个智能体的 `category` 字段，映射到新分类

**修改文件：`src/pages/Marketplace.tsx`**
- 更新 `categoryIcons` 映射

## 实现步骤

### Step 1: 修改 Agent 接口
