# 智能体创建向导优化计划

## 一、问题分析

### 1.1 当前存在的问题

| 问题 | 描述 |
|------|------|
| 技能与工具分离 | 只是简单勾选技能，没有与业务流程结合 |
| 缺少流程编排 | 没有可视化工作流设计器，无法定义执行顺序和逻辑 |
| 提示词设计简单 | 只有一个文本框，缺少结构化的提示词设计引导 |
| 缺少角色定义 | 没有明确智能体的角色定位和行为边界 |
| 缺少示例配置 | 没有Few-shot示例的配置入口 |

### 1.2 参考借鉴

**Coze (字节跳动)**:
- 可视化工作流编排
- 插件系统与流程节点结合
- 条件分支和循环节点

**Dify**:
- 结构化提示词设计
- 变量系统
- 工作流可视化

**FastGPT**:
- 知识库工作流
- 多模型编排

```
Step 1: 基础配置
├── 智能体名称
├── 描述
├── 分类选择
├── 图标选择
└── 高级设置（多轮对话、文件上传等）

├── 角色定义
│   ├── 角色名称
│   ├── 角色定位
│   └── 专业领域
├── 任务描述
│   ├── 核心任务
│   ├── 任务目标
│   └── 执行边界
├── 输出规范
│   ├── 输出格式
│   ├── 输出风格
│   └── 质量标准
├── 示例配置（Few-shot）
│   ├── 输入示例
│   └── 输出示例
└── 模型参数配置
    ├── Temperature
    ├── Max Tokens
    └── Top P

Step 3: 知识库配置
├── 选择知识库
├── 选择数据集
└── 检索设置

Step 4: 流程编排（核心重构）
├── 可视化工作流设计器
│   ├── 开始节点
│   ├── 输入处理节点
│   ├── 知识检索节点
│   ├── 模型推理节点
│   ├── 工具调用节点
│   ├── 条件分支节点
│   ├── 循环节点
│   └── 输出节点
├── 技能/工具配置（嵌入节点中）
│   ├── 输入技能（作为输入处理节点）
│   ├── 输出技能（作为输出节点）
│   └── 工具技能（作为工具调用节点）
└── 变量系统
    ├── 输入变量
    ├── 中间变量
    └── 输出变量

Step 5: 测试验证
├── 单元测试
├── 端到端测试
└── 边界测试

Step 6: 发布设置
├── 可见范围
├── 定价策略
├── 安全等级
└── 使用条款
```

### 3.2 Step 4: 流程编排

#### 节点类型定义

```typescript
interface WorkflowNode {
  id: string
  type: 'start' | 'input' | 'knowledge' | 'model' | 'tool' | 'condition' | 'loop' | 'output' | 'end'
  name: string
  config: Record<string, any>
  position: { x: number; y: number }
  connections: {
    source: string
    target: string
    condition?: string
  }[]
}
```

#### 节点详细设计

| 节点类型 | 图标 | 配置项 | 说明 |
|----------|------|--------|------|
| 开始节点 | ▶️ | 触发条件 | 定义智能体启动条件 |
| 输入处理节点 | 📥 | 输入类型、预处理逻辑 | 处理用户输入 |
| 知识检索节点 | 🔍 | 知识库选择、检索参数 | 检索相关知识 |
| 模型推理节点 | 🤖 | 模型选择、提示词模板 | LLM调用 |
| 工具调用节点 | 🔧 | 工具选择、参数配置 | 调用外部工具/API |
| 条件分支节点 | 🔀 | 条件表达式 | if-else逻辑 |
| 循环节点 | 🔄 | 循环条件、迭代变量 | 批量处理 |
| 输出节点 | 📤 | 输出格式、后处理 | 结果输出 |
| 结束节点 | ⏹️ | 结束状态 | 流程结束 |

#### 预置工作流模板

```
模板 1: 简单问答流程
开始 → 输入处理 → 知识检索 → 模型推理 → 输出 → 结束

模板 2: 文档生成流程
开始 → 输入处理 → 知识检索 → 模型推理 → 格式转换 → 输出 → 结束

模板 3: 多条件处理流程
开始 → 输入处理 → 条件分支 → [分支A: 知识检索] → 模型推理 → 输出 → 结束
                          → [分支B: 工具调用] → 模型推理 → 输出 → 结束

模板 4: 批量处理流程
开始 → 输入处理 → 循环节点 → [知识检索 → 模型推理] → 汇总 → 输出 → 结束
```

### 5.2 提示词生成逻辑

```typescript
function generateSystemPrompt(design: PromptDesign): string {
  return `
# 角色定义
你是${design.role.name}，${design.role.position}。
专业领域：${design.role.domains.join('、')}

# 任务说明
${design.task.coreTask}

任务目标：
${design.task.objectives.map(o => `- ${o}`).join('\n')}

执行边界：${design.task.boundaries}

# 输出规范
输出格式：${getFormatDesc(design.output.format)}
输出风格：${getStyleDesc(design.output.style)}
质量标准：${design.output.qualityStandards.join('、')}

# 示例
${design.examples.map(ex => `
输入：${ex.input}
输出：${ex.output}
`).join('\n')}
`.trim()
}
```
